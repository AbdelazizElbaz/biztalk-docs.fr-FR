# <a name="using-hl7-2x-schemas"></a>L’utilisation des schémas 2.X HL7
Cette section décrit les versions 2.X de la norme sept de niveau d’intégrité (HL7) pris en charge par [!INCLUDE[btsCoName](../../includes/btsconame-md.md)] [!INCLUDE[HL7_CurrentVersion_FirstRef](../../includes/hl7-currentversion-firstref-md.md)].  
  
> [!IMPORTANT]
>  Vous devrez peut-être mettre à jour les schémas installés avec BTAHL7 pour se conformer à la HL7 standard. Pour ce faire, consultez [résolution des erreurs de base de données](../../adapters-and-accelerators/accelerator-hl7/resolving-database-errors.md).  
  
> [!NOTE]
>  Le moteur de BTAHL7 ne peut pas traiter les instances de message conforme aux schémas HL7 qui ont une structure de schéma ambigu. Une structure de schéma ambigu est celle qui la norme HL7 n’a pas complètement défini. Ces schémas incluent ceux pour les types de messages CSU et SUR.
