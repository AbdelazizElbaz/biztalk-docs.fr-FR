# <a name="interact-adapter-non-repudiation"></a><span data-ttu-id="d15ad-102">Interagir carte de non-répudiation</span><span class="sxs-lookup"><span data-stu-id="d15ad-102">InterAct Adapter Non-Repudiation</span></span>
<span data-ttu-id="d15ad-103">Prise en charge de la non-répudiation d’un message interagir sortant est obtenu en définissant le SwInt:NRIndicator sur TRUE dans le SwInt:RequestControl ou SwInt:ResponseControl, selon le cas.</span><span class="sxs-lookup"><span data-stu-id="d15ad-103">Non-repudiation support for an outgoing InterAct message is obtained by setting the SwInt:NRIndicator to TRUE in the SwInt:RequestControl or SwInt:ResponseControl, as appropriate.</span></span> <span data-ttu-id="d15ad-104">Cela est nécessaire uniquement si le service ne sélectionne pas de prise en charge de la non répudiation par défaut, en fonction du profil de Service.</span><span class="sxs-lookup"><span data-stu-id="d15ad-104">This is required only if the service does not select non-repudiation support by default, according to the Service Profile.</span></span>  
  
 <span data-ttu-id="d15ad-105">Non répudiation est basée sur une signature valide créée par le SNL, juste avant la transmission du message.</span><span class="sxs-lookup"><span data-stu-id="d15ad-105">Non-repudiation is based on a valid signature created by the SNL, just before transmitting the message.</span></span> <span data-ttu-id="d15ad-106">Par conséquent, pour obtenir la prise en charge de la non répudiation sur les messages de demande sortants, il est obligatoire pour l’élément SwInt:RequestCrypto, contenu dans le SwInt:RequestControl, à définir la valeur TRUE dans le SwInt:RequestControl.</span><span class="sxs-lookup"><span data-stu-id="d15ad-106">Therefore, in order to obtain non-repudiation support on outgoing request messages, it is mandatory for the SwInt:RequestCrypto element, contained within the SwInt:RequestControl, to be set TRUE in the SwInt:RequestControl.</span></span>  
  
 <span data-ttu-id="d15ad-107">Pour les messages de réponse, l’exigence est équivalente, sauf que les éléments impliqués sont le SwInt:ResponseCrypto, contenue dans le SwInt:ResponseControl ; SwInt:ResponseCrypto doit être définie à TRUE.</span><span class="sxs-lookup"><span data-stu-id="d15ad-107">For response messages, the requirement is equivalent, except that the elements involved are the SwInt:ResponseCrypto, as contained within the SwInt:ResponseControl; SwInt:ResponseCrypto must be set TRUE.</span></span>  
  
 <span data-ttu-id="d15ad-108">Prise en charge de la non répudiation sur une demande, il est nécessaire pour la signature de message couvrir la SwInt:RequestHeader, SwInt:RequestPayload et le SwInt:SwiftRequestRef de la SwInt:RequestDescriptor.</span><span class="sxs-lookup"><span data-stu-id="d15ad-108">For non-repudiation support on a request, it is necessary for the message signature to cover the SwInt:RequestHeader, SwInt:RequestPayload, and the SwInt:SwiftRequestRef of the SwInt:RequestDescriptor.</span></span> <span data-ttu-id="d15ad-109">Le SwInt:SwiftRequestRef est généré automatiquement par SWIFTNet lien.</span><span class="sxs-lookup"><span data-stu-id="d15ad-109">The SwInt:SwiftRequestRef is automatically generated by SWIFTNet Link.</span></span> <span data-ttu-id="d15ad-110">Dans le cadre de la génération de la SwInt:SwiftRequestRef, le SNL ajuste aussi automatiquement la valeur de SwSec:MemberRef dans le SwSec:CryptoControl pour générer la signature de message requis.</span><span class="sxs-lookup"><span data-stu-id="d15ad-110">In connection with generating the SwInt:SwiftRequestRef, the SNL also automatically adjusts the SwSec:MemberRef value within the SwSec:CryptoControl for generating the required message signature.</span></span> <span data-ttu-id="d15ad-111">De même, pour repudiatin-non prise en charge sur une réponse, il est nécessaire pour la signature de message couvrir la SwInt:ResponseHeader, SwInt:ResponsePayload et le SwInt:SwiftResponseRef de la SwInt:ResponseDescriptor.</span><span class="sxs-lookup"><span data-stu-id="d15ad-111">Similarly, for non-repudiatin support on a Response, it is necessary for the message signature to cover the SwInt:ResponseHeader, SwInt:ResponsePayload, and the SwInt:SwiftResponseRef of the SwInt:ResponseDescriptor.</span></span> <span data-ttu-id="d15ad-112">Le SwInt:SwiftResponseRef est généré automatiquement par SWIFTNet lien.</span><span class="sxs-lookup"><span data-stu-id="d15ad-112">The SwInt:SwiftResponseRef is automatically generated by SWIFTNet Link.</span></span> <span data-ttu-id="d15ad-113">Dans le cadre de la génération de la SwInt:SwiftResponseRef, le SNL ajuste aussi automatiquement la valeur de SwSec:MemberRef dans le SwSec:CryptoControl pour générer la signature de message requis.</span><span class="sxs-lookup"><span data-stu-id="d15ad-113">In connection with generating the SwInt:SwiftResponseRef, the SNL also automatically adjusts the SwSec:MemberRef value within the SwSec:CryptoControl for generating the required message signature.</span></span>  
  
 <span data-ttu-id="d15ad-114">Si le profil de Service de l’entreprise sélectionne la non-répudiation par défaut, la signature de message nécessaire doit toujours être sélectionné (comme décrits ci-dessus, dans le SwInt:RequestControl ou le SwInt:ResponseControl) et doit être sélectionnée avant que le message quitte le SNL.</span><span class="sxs-lookup"><span data-stu-id="d15ad-114">If the business Service Profile selects non-repudiation by default, the necessary message signature is still required to be selected (as just described, in the SwInt:RequestControl or the SwInt:ResponseControl), and must be selected before the message leaves the SNL.</span></span>  
  
 <span data-ttu-id="d15ad-115">Si la sélection des fonctionnalités en fonction de l’entreprise de profil de Service appelle prise en charge de la non-répudiation d’un message, et si la signature nécessaire est introuvable avec le message, le message est rejeté par le commutateur.</span><span class="sxs-lookup"><span data-stu-id="d15ad-115">If Feature Selection according to the business Service Profile invokes Non-repudiation Support on a message, and if the necessary signature is not found with the message, then the message will be rejected by the Switch.</span></span> <span data-ttu-id="d15ad-116">Un message d’exception état s’affichera à l’expéditeur du message.</span><span class="sxs-lookup"><span data-stu-id="d15ad-116">A status exception message will be returned to the Sender of the message.</span></span>  
  
 <span data-ttu-id="d15ad-117">Chiffrement de la charge utile n’est pas cohérent avec la prise en charge de la Non répudiation.</span><span class="sxs-lookup"><span data-stu-id="d15ad-117">Payload encryption is not consistent with Non-repudiation Support.</span></span> <span data-ttu-id="d15ad-118">Si la prise en charge de la Non répudiation est activée pour un message et la charge utile est chiffrée en totalité ou en partie, le message est rejeté par SWIFTNet lien.</span><span class="sxs-lookup"><span data-stu-id="d15ad-118">If Non-repudiation Support is selected for a message and the payload is encrypted in whole or in part, then the message will be rejected by SWIFTNet Link.</span></span>  
  
 <span data-ttu-id="d15ad-119">Notez que si le service n’a pas la fonctionnalité de non-répudiation, toute demande ou réponse indiquant la non répudiation dans le contrôle, seront rejetées.</span><span class="sxs-lookup"><span data-stu-id="d15ad-119">Note that if the service does not have the non-repudiation feature, any request or response indicating non repudiation in the control, will be rejected.</span></span>
