# <a name="interact-adapter-end-to-end-reliable-delivery"></a><span data-ttu-id="3f376-102">Interagir remise fiable de bout en bout pour l’adaptateur</span><span class="sxs-lookup"><span data-stu-id="3f376-102">InterAct Adapter End-to-End Reliable Delivery</span></span>
<span data-ttu-id="3f376-103">Lors de l’envoi des messages ou des fichiers à un destinataire, nous vous recommandons de vous assurer que le fichier ou le message est remis, et que l’entreprise ou les transactions contenues dans ces sont exécutées plusieurs fois ne que prévu.</span><span class="sxs-lookup"><span data-stu-id="3f376-103">When sending messages or files to a recipient, we recommend that you ensure that the message or file is delivered, and that the business transaction(s) contained in these are executed no more times than anticipated.</span></span>  
  
 <span data-ttu-id="3f376-104">Lorsque les deux entités qui communiquent entre eux peuvent utiliser un stockage persistant (par exemple, fourni par un middleware orienté messages persistant et d’une application d’interface à l’utiliser), il est facile à implémenter une remise fiable si la méthode pour communiquer le état perçu du message est normalisé.</span><span class="sxs-lookup"><span data-stu-id="3f376-104">When both entities communicating with each other can use a persistent storage (for example, provided by a persistent message-oriented middleware and an interface application using it), it is easy to implement a reliable delivery if the way to communicate the perceived status of the message is standardized.</span></span>  
  
 <span data-ttu-id="3f376-105">L’illustration suivante montre un exemple de la structure de la E2EControl.</span><span class="sxs-lookup"><span data-stu-id="3f376-105">The following figure shows an example of the structure of the E2EControl.</span></span>  
  
 <span data-ttu-id="3f376-106">![Fin &#45; à &#45; fin du contrôle](../../adapters-and-accelerators/fileact-interact/media/63e39b43-118e-4572-9d75-21770253a1ee.gif "63e39b43-118e-4572-9d75-21770253a1ee")</span><span class="sxs-lookup"><span data-stu-id="3f376-106">![End&#45;to&#45;end control](../../adapters-and-accelerators/fileact-interact/media/63e39b43-118e-4572-9d75-21770253a1ee.gif "63e39b43-118e-4572-9d75-21770253a1ee")</span></span>  
  
 <span data-ttu-id="3f376-107">L’élément dans l’exemple illustré dans la figure est envoyé dans le SwInt:Request et remis inchangé dans la SwInt:RequestHandle à l’application réceptrice.</span><span class="sxs-lookup"><span data-stu-id="3f376-107">The element in the example shown in the figure is sent within the SwInt:Request and delivered unchanged within the SwInt:RequestHandle to the receiving application.</span></span> <span data-ttu-id="3f376-108">Ligne 02 permet d’affecter un identificateur unique à la demande.</span><span class="sxs-lookup"><span data-stu-id="3f376-108">Line 02 allows assigning a unique identifier to the request.</span></span> <span data-ttu-id="3f376-109">Cet identificateur unique est répété dans chaque retransmission suivante de la même demande.</span><span class="sxs-lookup"><span data-stu-id="3f376-109">This unique identifier is repeated in each subsequent re-transmission of the same request.</span></span>  
  
 <span data-ttu-id="3f376-110">La façon dont cet identificateur est construit est laissée à l’implémenteur, mais en général, il est basé sur un appel système tels que uuidgen(), ou il peut être le résultat de l’informatique un SHA-1 dans la demande à envoyer (avec Sw:MsgId préfixée et puis remplacez en base64 encodé s de SHA-1 chaîne).</span><span class="sxs-lookup"><span data-stu-id="3f376-110">The way this identifier is constructed is left to the implementer, but typically it is based on a system call such as uuidgen(), or it may be the result of computing a SHA-1 on the request to be sent (with a prefixed Sw:MsgId and then replace it by the base64 encoded SHA-1 string).</span></span> <span data-ttu-id="3f376-111">L’exigence principale est qu’il est globalement unique (avec une probabilité très élevée).</span><span class="sxs-lookup"><span data-stu-id="3f376-111">The main requirement is that it is globally unique (with a very high probability).</span></span>  
  
 <span data-ttu-id="3f376-112">Le Sw:CreationTime est l’heure de création de la demande d’origine.</span><span class="sxs-lookup"><span data-stu-id="3f376-112">The Sw:CreationTime is the time of creation of the original request.</span></span> <span data-ttu-id="3f376-113">Il s’agit d’un paramètre facultatif, mais il est utile de limiter les recherches éventuelles pour les tentatives précédentes de communication de ce message.</span><span class="sxs-lookup"><span data-stu-id="3f376-113">It is an optional parameter, but it is useful to limit eventual searches for previous communication attempts of this message.</span></span>  
  
 <span data-ttu-id="3f376-114">L’élément Sw:PDIndication est présente pour indiquer qu’il s’agit d’un deuxième ou le plus tentative de transmettre le message.</span><span class="sxs-lookup"><span data-stu-id="3f376-114">The element Sw:PDIndication is present to indicate that this is a second or further attempt to transmit the message.</span></span> <span data-ttu-id="3f376-115">L’application de réception qui tient compte de l’E2EControl puis permet le Sw:MsgId pour trouver précédent si le message a été reçu ou non.</span><span class="sxs-lookup"><span data-stu-id="3f376-115">The receiving application that is aware of the E2EControl can then use the Sw:MsgId to find back whether the message has been received or not.</span></span> <span data-ttu-id="3f376-116">L’option Sw:EmissionList contient l’heure des précédentes tentatives.</span><span class="sxs-lookup"><span data-stu-id="3f376-116">The optional Sw:EmissionList contains the time of the previous attempts.</span></span> <span data-ttu-id="3f376-117">Cette heure est l’heure locale de l’expéditeur (en temps universel) comme obtenu par l’expéditeur lors de l’utilisation de la fonction Sw:GetDateTime.</span><span class="sxs-lookup"><span data-stu-id="3f376-117">This time is the local time of the sender (in universal time) as got by the Sender when using the Sw:GetDateTime function.</span></span> <span data-ttu-id="3f376-118">Là encore, cela peut être utile de limiter les recherches.</span><span class="sxs-lookup"><span data-stu-id="3f376-118">Again this could be useful to limit searches.</span></span>
