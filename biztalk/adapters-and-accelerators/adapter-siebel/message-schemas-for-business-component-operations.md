# <a name="message-schemas-for-business-component-operations"></a>Schémas de message pour les opérations de composant d’entreprise
## <a name="message-schemas-for-siebel-business-component-operations"></a>Schémas de message pour les opérations de composant d’entreprise Siebel  
 Le tableau suivant répertorie les schémas de message pour les opérations exposées par le [!INCLUDE[adaptersiebel_short](../../includes/adaptersiebel-short-md.md)] sur les composants d’entreprise Siebel.  
  
|Opération|Structure XML| Description|  
|---------------|-------------------|-----------------|  
|Insert|Insérer le message :<br /><br /> `<Insert xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <ArrayOf[BC]InsertRecord>     <[BC]InsertRecord>       <[FIELD1_NAME]>value1</[FIELD1_NAME]>       <[FIELD2_NAME]>value2</[FIELD2_NAME]>       …     </[BC]InsertRecord>     …   </ArrayOf[BC]InsertRecord> </Insert>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [FIELD1_NAME] = nom de champ du composant entreprise ; par exemple, Account_x0020_Status.<br /><br /> Insérer un message de réponse :<br /><br /> `<InsertResponset xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <InsertResult>      <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[ID1]</string>      <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[ID2]</string>      …   </InsertResult> </InsertResponse>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 »<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [ID] = ID d’enregistrement ; par exemple, 42 6ODBL.|Insère un ou plusieurs enregistrements dans un composant d’entreprise.<br /><br /> La réponse contient les ID de tous les enregistrements sont insérés d’enregistrement.<br /><br /> Si au moins une insertion a réussi, puis aucune exception n’est levée et la réponse contient les ID des enregistrements a été insérés. Toutefois, si toutes les insertions échouent, une exception sera levée (XmlReaderParsingException ou TargetSystemException).|  
|Requête|Message de requête :<br /><br /> `<Query xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <ViewMode>[View_mode]</ViewMode>   <[BC]QueryInputRecord>     <SearchExpr>[Search_expr]</SearchExpr>     <SortSpec>Sort_spec</SortSpec>     <QueryFields>       <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[Field1_name]</string>       <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[Field2_name]</string>       …     </QueryFields>   </[BC]QueryInputRecord> </Query>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [View_mode] = mode d’affichage à appliquer. Il s'agit d'un paramètre facultatif. Consultez la documentation de Siebel pour obtenir une explication des valeurs possibles.<br /><br /> [Search_expr] = expression de recherche Siebel ; par exemple, [Name] comme « 3Com * ».<br /><br /> [Sort_spec] = la spécification de tri Siebel ; par exemple, nom (DESC), emplacement.<br /><br /> [FIELD1_NAME] = nom de champ du composant entreprise ; par exemple, état du compte.<br /><br /> Message de réponse de requête :<br /><br /> `<QueryResponse xmlns=" [VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <QueryResult>     <[BC]QueryRecord>       <[FIELD1_NAME]>value1</[FIELD1_NAME]>       <[FIELD2_NAME]>value2</[FIELD2_NAME]>       …     </[BC]QueryRecord >   </QueryResult> </QueryResponse>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [FIELD1_NAME] = nom de champ du composant entreprise ; par exemple, Account_x0020_Status.|Interroge un ou plusieurs enregistrements à partir du composant d’entreprise cible. L’opération de requête accepte les paramètres suivants :<br /><br /> \<SearchExpr > tous les enregistrements sous le composant de gestion cible sont comparés à l’expression de recherche et les enregistrements correspondants sont retournés.<br /><br /> \<SortSpec > la spécification de tri détermine l’ordre dans lequel les enregistrements qui correspondent à l’expression de recherche sont retournés. Ce paramètre est facultatif.<br /><br /> \<ChampsRequête > une liste de noms de champ qui spécifie les champs dans le composant d’entreprise cible qui doit être récupéré par l’opération de requête. Seuls les champs de cette liste sont renvoyés dans chaque enregistrement. Chaque champ doit être spécifié en utilisant le nom d’origine du champ dans le composant d’entreprise, pas le nom codé en XML ; par exemple, « Prénom » au lieu de « First_x0032_Name ». Ce paramètre est facultatif. Si les champs de la requête ne sont pas spécifiés, tous les champs sont retournés.|  
|Update|Message de mise à jour :<br /><br /> `<Update xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <ViewMode>[View_mode]</ViewMode>   <ArrayOf[BC]UpdateRecord>     <[BC]UpdateRecord>       <[FIELD1_NAME]>value1</[FIELD1_NAME]>       <[FIELD2_NAME]>value2</[FIELD2_NAME]>       …       <Id>[Record_ID]</Id>     </[BC]UpdateRecord>     …   </ArrayOf[BC]UpdateRecord> </Update>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [View_mode] = mode d’affichage à appliquer lors de l’interrogation des enregistrements qui doivent être mis à jour. Il s'agit d'un paramètre facultatif. Consultez la documentation de Siebel pour obtenir une explication des valeurs possibles.<br /><br /> [FIELD1_NAME] = nom de champ du composant entreprise ; par exemple, Account_x0020_Status.<br /><br /> [Record_ID] = ID d’enregistrement d’un enregistrement à mettre à jour ; par exemple, 42 60DBL.<br /><br /> Message de réponse de mise à jour :<br /><br /> `<UpdateResponse xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <UpdateResult>     <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[ID1]</string>     <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[ID2]</string>     …   </UpdateResult> </UpdateResponse>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [ID] = ID d’enregistrement ; par exemple, 42 6ODBL.|Mettre à jour un ou plusieurs enregistrements spécifiées par une liste d’enregistrements de mise à jour.<br /><br /> Chaque enregistrement de la mise à jour contient une liste des champs à mettre à jour et un seul, obligatoire \<Id > élément qui identifie l’enregistrement à mettre à jour.<br /><br /> La réponse contient une liste de l’enregistrement de l’ID de tous les enregistrements mis à jour.<br /><br /> Si au moins une mise à jour a réussi, puis aucune exception n’est levée et la réponse contient les ID des enregistrements de mise à jour réussie. Toutefois, si les mises à jour échouent, une exception sera levée (XmlReaderParsingException ou TargetSystemException).|  
|DELETE|Supprimer le message :<br /><br /> `<Delete xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <ViewMode>[View_mode]</ViewMode>   <Id>     <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[ID1]</string>     <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[ID2]</string>     …   </Id>   <SearchExpr>Search_expr</SearchExpr> </Delete>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [View_mode] = mode d’affichage à appliquer lors de l’interrogation des enregistrements qui doivent être supprimés. Il s'agit d'un paramètre facultatif. Consultez la documentation de Siebel pour obtenir une explication des valeurs possibles.<br /><br /> [ID] = ID d’enregistrement ; par exemple, 42 6ODBL.<br /><br /> Search_expr = expression de recherche Siebel ; par exemple, [Name] comme « 3Com * ».<br /><br /> Supprimer le message de réponse<br /><br /> `<DeleteResponse xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <DeleteResult>     <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[ID1]</string>     <string xmlns="http://schemas.microsoft.com/2003/10/Serialization/Arrays">[ID2]</string>     …   </DeleteResult> </DeleteResponse>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [ID] = ID d’enregistrement ; par exemple, 42 6ODBL.|Supprimer un ou plusieurs enregistrements correspondant à un ensemble d’ID d’enregistrement ou d’une expression de recherche.<br /><br /> **Important :** l’opération de suppression prend une expression de recherche ou d’un ensemble d’ID d’enregistrement. Vous ne pouvez pas spécifier les deux paramètres dans la même opération de suppression.<br /><br /> La réponse contient une liste de l’enregistrement ID de toutes les lignes supprimées.<br /><br /> Si au moins une suppression a réussi, puis aucune exception n’est levée et la réponse contient les ID des enregistrements supprimés avec succès. Toutefois, si toutes les suppressions échouent, une exception sera levée (XmlReaderParsingException ou TargetSystemException).|  
|Associer|Associer le message :<br /><br /> `<Associate xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <ViewMode>[View_mode]</ViewMode>   <ParentSearchExpr>Par_search</ParentSearchExpr>   <ParentMVGField>Field_name</ParentMVGField>   <ChildSearchExpr>Chld_search</ChildSearchExpr> </Associate>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [View_mode] = mode d’affichage à appliquer à la requête pour les enregistrements de composant d’entreprise parents et enfants. Si une valeur est spécifiée, cette valeur sera appliquée à la recherche de parent et la recherche d’enfants. Il s'agit d'un argument facultatif. Pour obtenir une explication des valeurs possibles, consultez la documentation de Siebel.<br /><br /> [Par_search] = expression de recherche correspondant à une seule activité composant parente ; par exemple, [Id] comme AB-12345.<br /><br /> [Chld_search] = expression de recherche correspondant à enregistrement de composant de professionnels exactement un seul enfant sera associé à l’enregistrement de composant parent entreprise ; par exemple, [Id] comme CD-12345.<br /><br /> [Nom_champ] = nom de champ de groupe à valeurs multiples du composant parent entreprise ; par exemple, « facture à prénom ' dans le composant de gestion de compte.<br /><br /> Associer un message de réponse :<br /><br /> `<AssociateResponse xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <AssociateResult>     <ChildID>[CHILD_ID]</ChildID>     <ParentID>[PARENT_ID]</ParentID>   </AssociateResult> </AssociateResponse>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [CHILD_ID] = ID d’enregistrement enfant ; par exemple, 42 6ODBL.<br /><br /> [PARENT_ID] = ID d’enregistrement Parent ; par exemple, 42 6ODBL.|L’opération d’association apparaissent uniquement pour les composants d’entreprise avec des champs de groupe à valeurs multiples (multiples). Il associe les enregistrements en spécifiant les expressions de recherche pour le parent et enfant.<br /><br /> L’opération d’association :<br /><br /> -Associe un enregistrement de composant d’entreprise parent sélectionné avec un enregistrement de composant d’activité enfant sélectionné<br /><br /> -Établit une association 1:1. un seul enregistrement parent à associer à l’enregistrement d’un enfant<br /><br /> -Les associations de 1:1 plusieurs doivent être effectuées pour une association de 1 à n, et plusieurs associations de 1 à n doivent être effectuées pour une association m : n<br /><br /> Remarque :<br /><br /> L’expression de recherche parent doit correspondre à un enregistrement unique dans la table parente.<br /><br /> L’expression de recherche enfant doit correspondre à un enregistrement unique dans la table enfant.<br /><br /> La valeur de retour contient l’ID d’enregistrement de la ligne enfant associée et l’ID d’enregistrement de la ligne parente associée. Si l’expression de recherche parent ou l’expression de recherche enfant retourne plus d’un enregistrement de correspondance, une exception est levée.|  
|Dissocier|Dissociation du message :<br /><br /> `<Dissociate xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <ViewMode>[View_mode]</ViewMode>   <ParentSearchExpr>Par_search</ParentSearchExpr>   <ParentMVGField>Field_name</ParentMVGField>   <ChildSearchExpr>Chld_search</ChildSearchExpr> </Dissociate>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [View_mode] = mode d’affichage à appliquer à la requête pour les enregistrements de composant d’entreprise parents et enfants. Si une valeur est spécifiée, cette valeur sera appliquée à la recherche de parent et la recherche d’enfants. Il s'agit d'un argument facultatif. Pour obtenir une explication des valeurs autorisées, consultez la documentation de Siebel.<br /><br /> [Par_search] = expression de recherche correspondant à une seule activité composant parente ; par exemple, [Id] comme AB-12345.<br /><br /> [Chld_search] = expression de recherche correspondant exactement une entreprise composant enregistrement enfant qui sera dissociée avec l’enregistrement de composant parent entreprise ; par exemple, [Id] comme CD-12345.<br /><br /> [Nom_champ] = fieldname de groupe à valeurs multiples du composant parent entreprise ; par exemple, « facture à prénom ' dans le composant de gestion de compte.<br /><br /> Dissociation du message de réponse :<br /><br /> `<DissociateResponse xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <DissociateResult>     <ChildID>[CHILD_ID]</ChildID>     <ParentID>[PARENT_ID]</ParentID>   </DissociateResult> </DisocciateResponse>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [CHILD_ID] = ID d’enregistrement enfant ; par exemple, 42 6ODBL.<br /><br /> [PARENT_ID] = ID d’enregistrement Parent ; par exemple, 42 6ODBL.|L’opération recréez-la apparaissent uniquement pour les composants d’entreprise avec des champs de groupe à valeurs multiples (multiples). Il dissocie les enregistrements en spécifiant les expressions de recherche pour le parent et enfant.<br /><br /> L’opération recréez-la :<br /><br /> Dissocie un enregistrement de composant parent sélectionné professionnels avec un enregistrement de composant d’activité enfant sélectionné<br /><br /> Établit une dissociation 1:1. un seul enregistrement parent sera dissocié avec l’enregistrement d’un enfant<br /><br /> Plusieurs dissociations de 1:1 doivent être effectuées pour une dissociation de 1 à n, et plusieurs dissociations de 1 à n doivent être effectuées pour une dissociation m : n<br /><br /> Notez que l’expression de recherche enfant doit correspondre à un enregistrement unique parmi les enregistrements associés avec les enregistrements parents qui correspond à l’expression de recherche du parent.<br /><br /> **Important :** il existe une différence dans l’expression de recherche enfant pour les opérations d’association et RECRÉEZ-la. Dans une opération d’association, l’expression de recherche enfant recherche tous les enregistrements dans le composant d’activité enfant. Dans une opération RECRÉEZ-la, l’expression de recherche enfant recherche uniquement les enregistrements dans le composant d’entreprise enfants qui sont associés à des enregistrements dans le composant parent de l’entreprise, où les enregistrements parents correspond à l’expression de recherche de parent.<br /><br /> La valeur de retour contient l’ID d’enregistrement de l’enregistrement enfant de dissociation et l’ID d’enregistrement de l’enregistrement parent de dissociation. Si l’expression de recherche parent ou l’expression de recherche enfant retourne plus d’un enregistrement de correspondance, une exception est levée.|  
|Query_ [MVG_Child_Business_Comp]|Message de Query_ [MVG_Child_Business_Comp] :<br /><br /> `<Query_[CHILD_BC] xmlns="BusinessObjects/[BO]/[BC]/Operation">   <ViewMode>View_mode</ViewMode>   <ParentSearchExpr>Par_search</ParentSearchExpr>   <ParentMVGField>Field_name</ParentMVGField>   <[CHILD_BC]QueryInputRecord>     <SearchExpr>Chld_search </SearchExpr>     <QueryFields>       <string>field1</string>       <string>field2</string>       …     </QueryFields>   </[CHILD_BC]QueryInputRecord> </Query_[CHILD_BC]>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [View_mode] = mode d’affichage à appliquer ; peut être de 0 à 9. Il s'agit d'un paramètre facultatif. Consultez la documentation de Siebel pour obtenir une explication de ces valeurs possibles.<br /><br /> [CHILD_BC] = nom du composant d’entreprise enfant associé au champ de groupe à valeurs multiples du composant parent entreprise ; par exemple, Contact (pour le composant de gestion de compte parent).<br /><br /> [Par_search] = expression de recherche de correspondance 1 enregistrement de composant d’entreprise parent ; par exemple, [Id] comme AB-12345.<br /><br /> [Nom_champ] = fieldname de groupe à valeurs multiples du composant Parent entreprise ; par exemple, « facture à prénom ' dans le composant de gestion de compte ».<br /><br /> [Chld_search] = expression de recherche correspondant à 1 ou plusieurs enfants composant documents professionnels ; par exemple, [Id] comme CD-12345.<br /><br /> [field1], [field2] et ainsi de suite = champs de la requête dans le composant d’activité enfant. Une liste de noms de champ qui spécifie les champs dans le composant d’entreprise enfant qui doit être récupéré par l’opération. Seuls les champs de cette liste sont renvoyés dans chaque enregistrement. Chaque champ doit être spécifié en utilisant le nom d’origine du champ dans le composant de gestion des enfants, pas le nom XML encodé ; par exemple, « Prénom » au lieu de « First_x0032_Name ». Ce paramètre est facultatif.<br /><br /> Message de réponse Query_ [MVG_Child_Business_Comp] :<br /><br /> `<Query_[CHILD_BC]Response xmlns="[VERSION]/BusinessObjects/[BO]/[BC]/Operation">   <Query_[CHILD_BC]Result>     <[CHILD_BC]QueryRecord>       <[FIELD1_NAME]>value1</[FIELD1_NAME]>       <[FIELD2_NAME]>value2</[FIELD2_NAME]>       …     </[CHILD_BC]QueryRecord >   </QueryResult> </QueryResponse>`<br /><br /> [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».<br /><br /> [BO] = nom de l’objet métier ; par exemple, le compte.<br /><br /> [BC] = nom du composant d’entreprise ; par exemple, le compte.<br /><br /> [FIELD1_NAME] = nom de champ du composant entreprise ; par exemple, Account_x0020_Status.<br /><br /> [CHILD_BC] = nom du composant d’entreprise enfant associé au champ de groupe à valeurs multiples du composant parent entreprise ; par exemple, Contact (pour le composant de gestion de compte parent).|Cette opération est exposée pour les composants d’entreprise avec des champs de groupe à valeurs multiples.<br /><br /> Il interroge les enregistrements enfants associés à un enregistrement parent qui est associé à un champ de groupe à valeurs multiples dans le composant d’entreprise parent.|  
  
## <a name="message-actions-for-siebel-business-component-operations"></a>Actions de message pour les opérations de composant d’entreprise Siebel  
 Le tableau suivant présente l’action SOAP pour chacune des opérations de composant d’entreprise Siebel. Seules les actions pour les messages de demande affichées, l’action pour le message de réponse est formé en ajoutant « / réponse » à l’action de message de demande. par exemple, « http://Microsoft.LobServices.Siebel/2007/03/BusinessObjects/Account/Account/Insert/response ».  
  
|Opération|Action|Exemple|  
|---------------|------------|-------------|  
|Insert|[VERSION] /BusinessObjects/ [BO] / [BC] / insertion|http://Microsoft.LobServices.Siebel/2007/03/BusinessObjects/Account/Account/INSERT|  
|Requête|[VERSION] /BusinessObjects/ [BO] / [BC] / requête|http://Microsoft.LobServices.Siebel/2007/03/BusinessObjects/Account/Account/Query|  
|Update|[VERSION] /BusinessObjects/ [BO] / [BC] / mise à jour|http://Microsoft.LobServices.Siebel/2007/03/BusinessObjects/Account/Account/Update|  
|DELETE|[VERSION] /BusinessObjects/ [BO] / [BC] / Delete|http://Microsoft.LobServices.Siebel/2007/03/BusinessObjects/Account/Account/Delete|  
|Associer|[VERSION] /BusinessObjects/ [BO] / [BC] / associer|http://Microsoft.LobServices.Siebel/2007/03/BusinessObjects/Account/Account/Associate|  
|Dissocier|[VERSION] /BusinessObjects/ [BO] / [BC] / dissocier|http://Microsoft.LobServices.Siebel/2007/03/BusinessObjects/Account/Account/Dissociate|  
|Query_ [CHILD_BC]|[VERSION] /BusinessObjects/ [BO] / [BC] / Query_ [CHILD_BC]|http://Microsoft.LobServices.Siebel/2007/03/BusinessObjects/Account/Account/Query_Contact|  
  
 [VERSION] = la chaîne de version de message ; par exemple, « http://Microsoft.LobServices.Siebel/2007/03 ».  
  
 [BO] = nom de l’objet métier ; par exemple, le compte.  
  
 [BC] = nom de composant d’entreprise ; par exemple, le compte.  
  
 Nom du composant d’entreprise enfant associé au champ de groupe à valeurs multiples du composant parent entreprise ; par exemple, Contact (pour le composant de gestion de compte parent).  
  
 Vous devez spécifier explicitement l’action du message lorsque vous consommez le [!INCLUDE[adaptersiebel_short](../../includes/adaptersiebel-short-md.md)] dans une solution BizTalk Server ou à l’aide de la [!INCLUDE[firstref_btsWinCommFoundation](../../includes/firstref-btswincommfoundation-md.md)] modèle de canal. Pour plus d’informations, consultez [développer vos applications Siebel](../../adapters-and-accelerators/adapter-siebel/develop-your-siebel-applications.md).
