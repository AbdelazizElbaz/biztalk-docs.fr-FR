## <a name="why-choose-the-wcf-service-model-or-the-wcf-channel-model"></a><span data-ttu-id="cc1ec-120">Pourquoi choisir le modèle de Service WCF ou le modèle de canal WCF ?</span><span class="sxs-lookup"><span data-stu-id="cc1ec-120">Why Choose the WCF Service Model or the WCF Channel Model?</span></span>  
 <span data-ttu-id="cc1ec-121">Car il présente un modèle qui est familier aux programmeurs .NET et masque la complexité sous-jacente d’échange de messages SOAP sur un canal, le modèle de service WCF est souvent le meilleur choix pour développer des solutions de programmation pour le [!INCLUDE[adaptersiebel_short](../../includes/adaptersiebel-short-md.md)].</span><span class="sxs-lookup"><span data-stu-id="cc1ec-121">Because it presents a model that is familiar to .NET programmers and hides the underlying complexities of SOAP message exchange over a channel, the WCF service model is often the best choice to develop programming solutions for the [!INCLUDE[adaptersiebel_short](../../includes/adaptersiebel-short-md.md)].</span></span> <span data-ttu-id="cc1ec-122">Toutefois, il existe des scénarios dans lesquels le modèle de canal WCF peut être un meilleur choix.</span><span class="sxs-lookup"><span data-stu-id="cc1ec-122">However, there are scenarios in which the WCF channel model might be a better choice.</span></span> <span data-ttu-id="cc1ec-123">Par exemple, sérialisation et désérialisation entre la représentation XML d’objets dans un message SOAP et les types .NET utilisés pour représenter dans le modèle de service WCF impliquent la lecture de la totalité du message en mémoire.</span><span class="sxs-lookup"><span data-stu-id="cc1ec-123">For example, serializing and de-serializing between the XML representation of objects in a SOAP message and the .NET types used to represent them in the WCF service model involves reading the entire message into memory.</span></span>  
  
 <span data-ttu-id="cc1ec-124">Le modèle de canal WCF prend en charge la diffusion en continu au niveau du nœud XML sur toutes les opérations.</span><span class="sxs-lookup"><span data-stu-id="cc1ec-124">The WCF channel model provides support for XML node-level streaming on all operations.</span></span> <span data-ttu-id="cc1ec-125">Dans le niveau de nœud de diffusion en continu, uniquement à chaque nœud du message XML est conservée en mémoire à tout moment.</span><span class="sxs-lookup"><span data-stu-id="cc1ec-125">In node-level streaming, only each node of the XML message is held in memory at any one time.</span></span> <span data-ttu-id="cc1ec-126">Pour certaines opérations, par exemple, si vous effectuez des requêtes qui retournent des jeux de résultats volumineux, le modèle de canal WCF peut être un meilleur choix pour votre application.</span><span class="sxs-lookup"><span data-stu-id="cc1ec-126">For certain operations, for example, if you are performing queries that return large result sets, the WCF channel model might be a better choice for your application.</span></span> <span data-ttu-id="cc1ec-127">Pour plus d’informations sur l’utilisation du modèle de canal WCF, consultez [développer les Applications Siebel à l’aide du modèle de canal WCF](../../adapters-and-accelerators/adapter-siebel/develop-siebel-applications-using-the-wcf-channel-model3.md).</span><span class="sxs-lookup"><span data-stu-id="cc1ec-127">For more information about using the WCF channel model, see [Develop Siebel Applications using the WCF Channel Model](../../adapters-and-accelerators/adapter-siebel/develop-siebel-applications-using-the-wcf-channel-model3.md).</span></span>
