## <a name="message-flow"></a><span data-ttu-id="1aa15-116">Flux de messages</span><span class="sxs-lookup"><span data-stu-id="1aa15-116">Message Flow</span></span>  
 <span data-ttu-id="1aa15-117">Le flux de messages à travers le processus privé du répondeur est comme suit :</span><span class="sxs-lookup"><span data-stu-id="1aa15-117">The message flow through the responder private process is as follows:</span></span>  
  
1.  <span data-ttu-id="1aa15-118">Le processus privé répondeur reçoit le message entrant d’origine du processus publics répondeur, dans l’itinéraire à partir de l’ordinateur de l’initiateur.</span><span class="sxs-lookup"><span data-stu-id="1aa15-118">The responder private process receives the original incoming message from the responder public process, in route from the initiator computer.</span></span>  
  
2.  <span data-ttu-id="1aa15-119">Le processus privé construit le message de l’application métier.</span><span class="sxs-lookup"><span data-stu-id="1aa15-119">The private process constructs the LOB application message.</span></span> <span data-ttu-id="1aa15-120">Cela implique la mise en route le modèle de message LOB, sérialise le contenu de service XML et le chargement les parties de message XML dans le message LOB.</span><span class="sxs-lookup"><span data-stu-id="1aa15-120">This involves getting the LOB message template, serializing the XML service content, and loading the XML message parts into the LOB message.</span></span>  
  
3.  <span data-ttu-id="1aa15-121">Le processus privé Achemine le message dans la table MessagesFromLOB de la [!INCLUDE[btaBTARN3.3abbrevnonumber](../../includes/btabtarn3-3abbrevnonumber-md.md)]données [!INCLUDE[btsSQLServerNoVersion](../../includes/btssqlservernoversion-md.md)] base de données, dans l’itinéraire à l’application métier principale.</span><span class="sxs-lookup"><span data-stu-id="1aa15-121">The private process routes the message to the MessagesFromLOB table in the [!INCLUDE[btaBTARN3.3abbrevnonumber](../../includes/btabtarn3-3abbrevnonumber-md.md)]DATA [!INCLUDE[btsSQLServerNoVersion](../../includes/btssqlservernoversion-md.md)] database, in route to the back-end LOB application.</span></span>  
  
4.  <span data-ttu-id="1aa15-122">Si le message d’origine a une pièce jointe, le processus privé appelle le composant AttachmentHelper pour traiter la pièce jointe.</span><span class="sxs-lookup"><span data-stu-id="1aa15-122">If the original message has an attachment, the private process calls the AttachmentHelper component to process the attachment.</span></span>  
  
5.  <span data-ttu-id="1aa15-123">Le processus privé envoie une notification à l’application métier qu’il enregistré le message de réponse dans la table MessagesToLOB.</span><span class="sxs-lookup"><span data-stu-id="1aa15-123">The private process sends a notification to the LOB application that it saved the response message in the MessagesToLOB table.</span></span>  
  
6.  <span data-ttu-id="1aa15-124">Si le message est un message d’action unique, le processus privé est terminé.</span><span class="sxs-lookup"><span data-stu-id="1aa15-124">If the message is a single-action message, the private process is complete.</span></span>  
  
7.  <span data-ttu-id="1aa15-125">Si le message est un message de double action, le processus privé attend une réponse de l’application métier.</span><span class="sxs-lookup"><span data-stu-id="1aa15-125">If the message is a double-action message, the private process listens for a response from the LOB application.</span></span>  
  
8.  <span data-ttu-id="1aa15-126">Lorsque le processus privé reçoit la réponse de l’application métier, il génère un message de réponse et envoie le message vers le processus public.</span><span class="sxs-lookup"><span data-stu-id="1aa15-126">When the private process receives the response from the LOB application, it constructs a response message, and sends the message to the public process.</span></span>  
  
9. <span data-ttu-id="1aa15-127">Le processus privé attend le signal dans le processus public.</span><span class="sxs-lookup"><span data-stu-id="1aa15-127">The private process waits for the signal from the public process.</span></span> <span data-ttu-id="1aa15-128">S’il reçoit le signal, il construit le message de signal LOB et l’envoie à l’application métier.</span><span class="sxs-lookup"><span data-stu-id="1aa15-128">If it receives the signal, it constructs the LOB signal message and sends it to the LOB application.</span></span> <span data-ttu-id="1aa15-129">Si la version RNIF est 1.1, le processus privé sera d’écoute pour un signal d’accusé de réception deuxième et à sa réception, de construire le message de signal LOB et envoyer à l’application métier.</span><span class="sxs-lookup"><span data-stu-id="1aa15-129">If the RNIF version is 1.1, the private process will listen for a second acknowledgement signal, and upon receiving it, will construct the LOB signal message and send it to the LOB application.</span></span> <span data-ttu-id="1aa15-130">Le processus privé notifie l’application métier après l’envoi de chaque message de signal.</span><span class="sxs-lookup"><span data-stu-id="1aa15-130">The private process notifies the LOB application after sending each signal message.</span></span>  
  
10. <span data-ttu-id="1aa15-131">Si le processus privé reçoit un message de Notification d’échec (n) à partir du processus public, dans l’itinéraire à partir de l’initiateur, le processus privé construit une «[!INCLUDE[btaBTARN3.3abbrevnonumber](../../includes/btabtarn3-3abbrevnonumber-md.md)]Exception », le message et envoie à l’application métier.</span><span class="sxs-lookup"><span data-stu-id="1aa15-131">If the private process receives a Notification of Failure (NoF) message from the public process, in route from the initiator, the private process constructs a "[!INCLUDE[btaBTARN3.3abbrevnonumber](../../includes/btabtarn3-3abbrevnonumber-md.md)]Exception" message, and sends it to the LOB application.</span></span>
