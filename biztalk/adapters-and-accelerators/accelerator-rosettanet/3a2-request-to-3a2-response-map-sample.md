# <a name="3a2-request-to-3a2-response-map-sample"></a>Demande de 3A2 d’exemple de mappage de réponse 3A2
L’exemple _3A2RequestMessageTo3A2ResponseMessage.btm montre comment vous pouvez mapper un message de demande 3A2 à un message de réponse 3A2.  
  
 Par défaut, le [!INCLUDE[btsCoName](../../includes/btsconame-md.md)]® programme d’installation de BizTalk Server installe l’exemple dans \< *lecteur*\>: \Program Files\\ [!INCLUDE[btsCoName](../../includes/btsconame-md.md)] Microsoft BizTalk \<version \> Accelerator for RosettaNet\SDK\PIPAutomation\DoubleAction.  
  
## <a name="sample-contents"></a>Exemple de contenu  
 Cet exemple montre comment mapper les champs d’un message de demande 3A2 à un message de réponse 3A2. Vous pouvez utiliser cet exemple avec l’exemple d’Orchestration Double Action PIPAutomation. L’exemple d’Orchestration Double Action PIPAutomation montre comment implémenter une orchestration pour générer automatiquement des réponses pour l’action de double 0C 2, 0c4, 3A2 et 3 a 4 PIP.
