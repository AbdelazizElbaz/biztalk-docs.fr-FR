# <a name="proactivity-in-biztalk-server"></a>Proactivité dans BizTalk Server
Article de la technique de BizTalk  
  
 **Proactivité dans BizTalk Server**  
  
 **Enregistreur :** Tord féliciterez de l’avoir Nordahl (Bouvet AS)  
  
 **Réviseurs techniques :** Sandro Pereira (Devscope), Jan Steef Wiggers (Motion10), Erik Thue (Bouvet AS), Alexander Thue (Bouvet AS), Saravana Kumar (BizTalk360), Mandi Ohlinger (Microsoft)  
  
 **Date de publication :** juillet 2013  
  
 **S’applique à :** BizTalk Server  
  
 **Résumé :** le livre blanc s’applique à partir de BizTalk Server 2004 dans les environnements de BizTalk Server 2013.  L’objectif est de fournir des indications de façon proactive et résoudre ou éviter les problèmes potentiels avant qu’ils surviennent. Elle s’applique à tous les types d’utilisateurs de BizTalk, y compris les architectes, les développeurs et administrateurs. Pas tous les scénarios décrits peuvent se produire dans votre environnement et certains scénarios non mentionnés dans ce livre blanc peuvent se produire.  
  
 En raison de la nature et l’importance de BizTalk Server, les paramètres et les meilleures pratiques peuvent varier selon l’environnement ; y compris les paramètres réseau, conception de l’architecture, des versions, des flux de messages et des ressources. Ce livre blanc décrit le programme d’installation courants et les valeurs recommandées.
