# <a name="checklist-configuring-windows-server"></a>Liste de vérification : Configuration de Windows Server
Cette rubrique répertorie les étapes à suivre lors de la préparation de Windows Server pour une utilisation dans une production [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] environnement.  
  
##  <a name="BKMK_Win2k8"></a>Configurer Windows Server  
  
|Étapes|Référence|  
|-----------|---------------|  
|Configurer MSDTC pour [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)].|-Découvrez [comment activer MSDTC sur le serveur BizTalk](http://go.microsoft.com/fwlink/?LinkId=153236) (http://go.microsoft.com/fwlink/?LinkId=153236).<br />-Découvrez [résolution des problèmes liés à MSDTC](http://go.microsoft.com/fwlink/?LinkId=153237) (http://go.microsoft.com/fwlink/?LinkId=153237).|  
|Configurer l’ou les pare-feu pour [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]. **Remarque :** cette étape est uniquement requis si un ou plusieurs pare-feu sont en place dans votre [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] environnement.|-Découvrez [Ports requis pour BizTalk Server](http://go.microsoft.com/fwlink/?LinkId=153238) (http://go.microsoft.com/fwlink/?LinkId=153238).<br />-Consultez l’article de la Base de connaissances Microsoft 154596, [« Comment configurer l’allocation de port dynamique RPC avec un pare-feu »](http://go.microsoft.com/fwlink/?LinkId=153239) (http://go.microsoft.com/fwlink/?LinkId=153239).|  
|Désactiver l’hyperthreading sur tous les ordinateurs exécutant [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] et [!INCLUDE[btsSQLServerNoVersion](../includes/btssqlservernoversion-md.md)] dans le [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] environnement.|-Il est essentiel que l’hyperthreading est désactivé pour les ordinateurs exécutant [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]. Il s’agit d’un paramètre du BIOS, que se trouve généralement dans les paramètres de processeur de la configuration du BIOS. L’hyperthreading rend le serveur semble avoir davantage de cœurs de processeurs/processeur qu’il le fait ; Toutefois, les processeurs hyperthreaded fournissent en général entre 20 et 30 % des performances d’un cœur de processeur/processeur physique. Lorsque [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] nombres le nombre de processeurs pour ajuster ses algorithmes de réglage automatique, les processeurs hyperthreaded provoque ces réglages pour être dévié, c'est-à-dire nuire aux performances globales.<br />-L’Hyperthreading doit être désactivée pour [!INCLUDE[btsSQLServerNoVersion](../includes/btssqlservernoversion-md.md)] ordinateurs, car les applications qui peuvent provoquer des niveaux élevés de contention (tel que [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)]) peut provoquer une diminution des performances dans un environnement multithread sur un [!INCLUDE[btsSQLServerNoVersion](../includes/btssqlservernoversion-md.md)] ordinateur.|  
|Vérifiez la planification de processeur Windows Server est définie sur « Services d’arrière-plan ».|En garantissant que la valeur de cette option de configuration sur tous les ordinateurs exécutant Windows Server dans votre environnement améliorera les performances globales du système. Suivez ces étapes pour vous assurer de que Windows Server est configuré pour favoriser les services d’arrière-plan :<br /><br /> 1.  Cliquez sur **Démarrer**, cliquez sur **exécuter**, puis tapez **sysdm.cpl** dans les **exécuter** boîte.<br />2.  Dans le **propriétés système** boîte de dialogue, cliquez sur le **avancé** onglet, puis cliquez sur **paramètres** sous **performances**.<br />3.  Dans le **les Options de performances** boîte de dialogue, cliquez sur le **avancé** onglet, vérifiez que le **services d’arrière-plan** option est sélectionnée sous **processeur planification**, cliquez sur **OK**, puis cliquez sur **OK** à nouveau pour fermer **propriétés système** boîte de dialogue.|  
|Placez le fichier de pagination Windows sur un disque physique local séparé.|Déplacer le fichier d’échange sur un volume physique distinct autre que le système d’exploitation sur un ordinateur exécutant Windows Server améliore les performances en réduisant les contentions de disque.<br /><br /> Suivez ces étapes pour déplacer le fichier d’échange sur un volume physique distinct autre que le système d’exploitation :<br /><br /> 1.  Cliquez sur **Démarrer**, cliquez sur **exécuter**, puis tapez **sysdm.cpl** dans les **ouvrir** boîte.<br />2.  Cliquez sur le **avancé** onglet, puis cliquez sur **paramètres** sous **performances**.<br />3.  Cliquez sur le **avancé** , cliquez sur **modification** sous **mémoire virtuelle**, spécifiez les options pour le fichier d’échange, cliquez sur **OK** puis cliquez sur **OK** à nouveau pour fermer les propriétés système. **Remarque :** vous devez redémarrer l’ordinateur pour les nouveaux paramètres prennent effet.|  
|-Défragmentation de tous les disques (local et SAN/NAS) régulièrement par la planification de la défragmentation de disque heures creuses.<br />-Défragmenter le fichier de pagination Windows et allouer à l’avance les Tables du fichier maître de chaque disque le [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] environnement pour optimiser les performances globales du système.|Utilisez le [utilitaire PageDefrag](http://go.microsoft.com/fwlink/?LinkId=108976) (http://go.microsoft.com/fwlink/?LinkId=108976) pour défragmenter le fichier de pagination Windows et allouer à l’avance les Tables de fichiers principal.|  
|Si un logiciel antivirus est installé sur l’ordinateur exécutant [!INCLUDE[btsSQLServerNoVersion](../includes/btssqlservernoversion-md.md)], désactivez l’analyse en temps réel des données et transaction des fichiers (.mdf, .ndf, .ldf, .mdb).|Analyse en temps réel de la [!INCLUDE[btsSQLServerNoVersion](../includes/btssqlservernoversion-md.md)] les fichiers de données et des transactions peuvent augmenter les conflits d’e/s disque et réduire [!INCLUDE[btsSQLServerNoVersion](../includes/btssqlservernoversion-md.md)] performances.|  
|Si un logiciel antivirus est installé sur l’ordinateur exécutant [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)], désactivez l’analyse en temps réel des types de fichier non exécutable référencés par aucune [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] (en général les emplacements de réception. XML, mais peut également être .csv, .txt, etc..).|Analyse en temps réel des fichiers non exécutable qui sont référencées par [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] réception emplacements peuvent augmenter les conflits d’e/s sur ces fichiers et réduire [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] performances.|  
|Si un logiciel de détection des intrusions est installé, désactiver réseau entre des ordinateurs exécutant d’analyse [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] et des référentiels de données externes ([!INCLUDE[btsSQLServerNoVersion](../includes/btssqlservernoversion-md.md)]) ou les services (par exemple, Message Queuing et WebSphere MQSeries) de messagerie.|Logiciel de détection des intrusions peut ralentir ou même empêcher les communications valides sur le réseau.|  
|Les pilotes de carte (NIC) sur tous les ordinateurs du réseau le [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] environnement doit être réglé pour les performances.|Ajustez les pilotes de périphérique réseau pour optimiser la quantité de mémoire disponible pour la mise en mémoire tampon de paquet, entrante et sortante. Aussi optimiser le nombre de mémoires tampons, en particulier tampons de transmission et fusionner les tampons. La valeur par défaut les valeurs pour ces paramètres et si elles sont encore fournies, varier entre les versions de pilotes et les fabricants. L’objectif est d’optimiser le travail effectué par le matériel de carte d’interface réseau et pour permettre l’espace de mémoire tampon possible plus grande pour les opérations de réseau atténuer les pics de trafic réseau et de congestion associée.|  
|Définir les cartes réseau à une vitesse fixe et le mode duplex|Utiliser une vitesse fixe et le mode duplex (1 Gigabit ou plus avec duplex intégral) pour les connexions réseau sur les serveurs BizTalk et SQL. Cela garantit que l’interface réseau ne pas négocie automatiquement une vitesse inférieure ou le paramètre duplex, ce qui a été un problème avec certains commutateurs de l’entreprise dans le passé. Également, dans les environnements à volume élevé, il est conseillé de posséder des réseaux Gigabit.|  
|Arrêtez ou désactivez tous les services Windows qui ne sont pas strictement nécessaires (par exemple, le spouleur d’impression et le Service d’indexation) sur tous les ordinateurs dans le [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] environnement.|Les services inutiles en cours d’exécution sur un serveur de production utilise les ressources système qui pourrait être utilisées par [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] ou [!INCLUDE[btsSQLServerNoVersion](../includes/btssqlservernoversion-md.md)].|
