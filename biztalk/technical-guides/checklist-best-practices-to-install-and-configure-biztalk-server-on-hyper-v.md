# <a name="checklist-best-practices-for-installing-and-configuring-biztalk-server-on-hyper-v"></a>Liste de vérification : Meilleures pratiques pour l’installation et la configuration de BizTalk Server sur Hyper-V
Les sections ci-dessous présentent un résumé de l’installation et la configuration requise décrite dans le [déploiement BizTalk Server sur Hyper-V](../technical-guides/deploying-biztalk-server-on-hyper-v.md) section de ce guide. Ceux-ci doivent être utilisés comme une référence rapide lors de l’installation, configuration et déploiement [!INCLUDE[btsBizTalkServerNoVersion](../includes/btsbiztalkservernoversion-md.md)] dans un environnement Hyper-V. Pour plus d’informations, vous trouverez des liens vers les sections appropriées.  
  
## <a name="before-installing-hyper-v"></a>Avant d’installer Hyper-V...  
  
|Étape|Référence|  
|----------|---------------|  
|Hyper-V est un rôle de serveur pour les éditions 64 bits de Windows Server.|Consultez [liste de vérification : meilleures pratiques pour l’installation et la configuration de BizTalk Server sur Hyper-V](../technical-guides/checklist-best-practices-to-install-and-configure-biztalk-server-on-hyper-v.md)|  
|Assurez-vous que votre processeur prend en charge la virtualisation à assistance matérielle et la prévention de l’exécution des données (PED), et que ces fonctionnalités sont activées. Cela nécessite un processeur qui est compatible avec la technologie de virtualisation Intel (VT Intel) ou AMD Virtualization (AMD-V).|Consultez [installer le rôle Hyper-V](https://docs.microsoft.com/windows-server/virtualization/hyper-v/get-started/install-the-hyper-v-role-on-windows-server).|  
|Utiliser Windows Server Core Edition pour la partition racine. Cela réduire la charge sur le serveur et améliorer les performances de Hyper-V.|Consultez [installation Server Core](https://docs.microsoft.com/windows-server/get-started/getting-started-with-server-core).|  
|Exécutez uniquement le rôle de serveur Hyper-V sur la partition racine.|Consultez [serveurs Hyper-V de réglage des performances](https://docs.microsoft.com/windows-server/administration/performance-tuning/role/hyper-v-server/):<br />**Rôle de serveur dédié**<br />La partition racine doit être dédiée au rôle de serveur de virtualisation. Rôles serveur supplémentaires peuvent nuire aux performances du serveur de virtualisation, surtout si elles consomment beaucoup de bande passante processeur, mémoire ou e/s. En réduisant les rôles de serveur dans la partition racine a des avantages supplémentaires tels que la réduction de la surface d’attaque et la fréquence des mises à jour. Les administrateurs système prenez soin de logiciels installés dans la partition racine, car certains logiciels peuvent affecter négativement les performances globales du serveur de virtualisation.<br/><br/> Consultez [Configuration Hyper-V](https://docs.microsoft.com/windows-server/administration/performance-tuning/role/hyper-v-server/configuration) pour obtenir des conseils.|  
  
## <a name="when-creating-hyper-v-virtual-machines"></a>Lors de la création d’ordinateurs virtuels Hyper-V...  
  
|Étape|Référence|  
|----------|---------------|  
|À l’aide d’un disque dur virtuel de taille fixe (VHD) fournit des performances améliorées par rapport à redimensionné dynamiquement des disques durs virtuels pour les lecteurs de système d’exploitation.|Consultez [performances d’e/s de stockage Hyper-V](https://docs.microsoft.com/en-us/windows-server/administration/performance-tuning/role/hyper-v-server/storage-io-performance) pour obtenir des instructions : <br />**Disque dur virtuel de taille fixe**<br />Espace du disque dur virtuel est d’abord alloué lorsque le fichier de disque dur virtuel est créé. Ce type de disque dur virtuel est moins apt fragmenter, ce qui réduit le débit d’e/s lors d’une seule e/s est fractionnée en plusieurs e/s. Il a la surcharge du processeur la plus faible des trois types de disque dur virtuel, car les lectures et écritures en n’avez pas besoin rechercher le mappage du bloc.|  
|Utiliser des disques de taille fixe disque dur virtuel (VHD) pour les activités d’e/s de disque élevée et configurer les disques pour les volumes de données à l’aide du contrôleur SCSI. Pour les charges d’e/s de stockage très importantes qui s’étendent sur plusieurs lecteurs de données, chaque disque dur virtuel doit être attaché à un contrôleur SCSI synthétique distinct pour meilleures performances globales. En outre, chaque disque dur virtuel doit être stocké sur des disques physiques distincts.|Consultez [performances d’e/s de stockage Hyper-V](https://docs.microsoft.com/en-us/windows-server/administration/performance-tuning/role/hyper-v-server/storage-io-performance) pour obtenir des instructions :<br />**Contrôleur SCSI synthétique**<br />Le contrôleur de stockage synthétique fournit améliorer considérablement les performances sur les e/s de stockage avec la surcharge du processeur réduite que le périphérique IDE émulé. Les services d’intégration VM incluent le pilote compatible pour ce périphérique de stockage et sont requis pour le système d’exploitation invité détecte. Le disque du système d’exploitation doit être monté sur le périphérique IDE pour le système d’exploitation démarre correctement, mais les services d’intégration VM chargement un pilote de filtre qui redirige le périphérique IDE e/s au périphérique de stockage synthétique.<br />Nous vous recommandons vivement de montage de lecteurs de données directement au contrôleur SCSI synthétique, car cette configuration a réduit la surcharge du processeur. Vous devez également monter des fichiers journaux et le fichier d’échange de système d’exploitation directement au contrôleur SCSI synthétique si leur taux d’e/s attendu est élevé.<br />Pour les charges d’e/s de stockage très importantes qui s’étendent sur plusieurs lecteurs de données, chaque disque dur virtuel doit être attaché à un contrôleur SCSI synthétique distinct pour meilleures performances globales. En outre, chaque disque dur virtuel doit être stocké sur des disques physiques distincts.|  
|Pour attacher des disques de disque dur virtuel pour les activités d’e/s élevées, telles que les fichiers journaux et données SQL Server, utilisez le contrôleur SCSI. **Remarque :** n’attachez pas un disque du système à un contrôleur SCSI. Un disque dur virtuel qui contient un système d’exploitation doit être attaché à un contrôleur IDE.|Bien que le contrôleur IDE de Hyper-V et contrôleur SCSI offrent des performances comparables, le contrôleur SCSI peut être installé uniquement si les services d’intégration Hyper-V sont installés. Par conséquent, l’utilisation du contrôleur SCSI à attacher des disques directs garantit que les services d’intégration Hyper-V sont installés garantissant à son tour performances d’e/s disque optimales.|  
|Utiliser la carte réseau au lieu de la carte réseau virtuelle héritée lors de la configuration de mise en réseau pour un ordinateur virtuel. La carte réseau héritée est conçue pour les systèmes d’exploitation qui ne prennent pas en charge les composants d’intégration.|**Carte réseau synthétique**<br />Fonctionnalités d’Hyper-V une carte réseau synthétique qui est conçue spécifiquement pour les machines virtuelles atteindre considérablement réduite surcharge du processeur sur les e/s réseau lorsqu’elle est comparée à la carte réseau émulée qui reproduit le matériel existant. La carte réseau synthétique communique entre les partitions enfants et racine sur VMBus à l’aide de la mémoire partagée pour le transfert de données plus efficace. La carte réseau émulée doit être supprimée via la boîte de dialogue des paramètres de machine virtuelle et remplacée par une carte réseau synthétique. L’invité nécessite que les services d’intégration VM est installé.|  
|Assurez-vous que les services d’intégration sont installés sur tous les systèmes d’exploitation invités compatibles et vérifiez que la version la plus récente d’integration services est installée. Pour vérifier la version la plus récente d’integration services, exécutez **mise à jour Windows**.|Consultez [les performances du processeur Hyper-V](https://docs.microsoft.com/windows-server/administration/performance-tuning/role/hyper-v-server/processor-performance) pour obtenir des instructions :<br />**Invités compatibles**<br />Il peut être recommandé d’utiliser Windows Server comme système d’exploitation invité. Les enlightments réduire la surcharge du processeur de Windows qui s’exécute dans une machine virtuelle. Les services d’intégration fournissent enlightments supplémentaires pour les e/s. En fonction de la charge du serveur, il peut être approprié héberger une application serveur sur un invité Windows Server pour de meilleures performances.|  
|Chaque fois que possible, configurez une allocation de 1-1 de processeurs virtuels pour les processeurs logiques disponibles.|Pour plus d’informations sur la configuration d’une allocation de 1-à-1 de processeurs virtuels pour les processeurs logiques disponibles, consultez la section « Optimiser les performances du processeur » dans [liste de vérification : optimisation des performances sur Hyper-V](~/technical-guides/checklist-optimizing-performance-on-hyper-v.md).|  
|Convertir ou migrer des ordinateurs virtuels en cours d’exécution sur Microsoft Virtual PC, Microsoft Virtual Server ou VMWare ESX Server pour s’exécuter sur Hyper-V.|-Utilisez System Center Virtual Machine Manager convertir ou migrer des ordinateurs virtuels à exécuter sur Hyper-V. <br />-Si nécessaire, le processus de conversion d’ordinateurs virtuels en cours d’exécution sur Microsoft Virtual PC ou Microsoft Virtual Server peut être effectué manuellement. Pour plus d’informations, consultez [Guide de Migration de Machine virtuelle : comment pour migrer de Virtual Server vers Hyper-V](http://go.microsoft.com/fwlink/?LinkID=137258).<br />-L’outil exemple **VMC2Hyper-V** peut également être utilisé pour migrer des ordinateurs virtuels en cours d’exécution sur Microsoft Virtual PC ou Microsoft Virtual Server vers Hyper-V. Pour plus d’informations sur l’exemple d’outil VMC2Hyper-V, consultez [l’ordinateur virtuel Hyper-V importation outil disponible](http://go.microsoft.com/fwlink/?LinkID=135683). |
