## <a name="improving-performance-with-nested-scopes"></a><span data-ttu-id="ed70f-150">Amélioration des performances avec des étendues imbriquées</span><span class="sxs-lookup"><span data-stu-id="ed70f-150">Improving Performance with Nested Scopes</span></span>  
 <span data-ttu-id="ed70f-151">Un important de noter que sur le **OrderBroker** orchestration est son utilisation des étendues imbriquées.</span><span class="sxs-lookup"><span data-stu-id="ed70f-151">One of the noticeable things about the **OrderBroker** orchestration is its use of nested scopes.</span></span> <span data-ttu-id="ed70f-152">Les étendues imbriquées sont là, notamment, pour améliorer les performances en limitant les points de persistance.</span><span class="sxs-lookup"><span data-stu-id="ed70f-152">The nested scopes are there, in part, to improve performance by limiting the persistence points.</span></span>

 <span data-ttu-id="ed70f-156">Avec cinq **envoyer** formes, le **OrderBroker** orchestration doit avoir cinq points de persistance.</span><span class="sxs-lookup"><span data-stu-id="ed70f-156">With five **Send** shapes, the **OrderBroker** orchestration should have five persistence points.</span></span> <span data-ttu-id="ed70f-157">Toutefois, lorsque vous regroupez **envoyer** formes à l’intérieur d’une étendue de transaction atomique, le moteur reconnaît qu’il doit uniquement un point de persistance pour l’étendue.</span><span class="sxs-lookup"><span data-stu-id="ed70f-157">However, when you group **Send** shapes inside an atomic transaction scope, the engine recognizes it only needs one persistence point for the scope.</span></span> <span data-ttu-id="ed70f-158">Étant donné que quatre le **envoyer** en formes **OrderBroker** orchestration ne font pas partie des gestionnaires d’exceptions et aucune opération ne doit être effectuée après l’envoi, ils peuvent entrer dans une étendue de transaction atomique.</span><span class="sxs-lookup"><span data-stu-id="ed70f-158">Because four of the **Send** shapes in **OrderBroker** orchestration are not part of exception handlers and nothing needs to be done after the send, they can go in an atomic transaction scope.</span></span> <span data-ttu-id="ed70f-159">Cela réduit le nombre de points de persistance.</span><span class="sxs-lookup"><span data-stu-id="ed70f-159">This reduces the number of persistence points.</span></span> <span data-ttu-id="ed70f-160">Pour plus d’informations sur les transactions atomiques, consultez [Transactions atomiques](../core/atomic-transactions.md).</span><span class="sxs-lookup"><span data-stu-id="ed70f-160">For more information about atomic transactions, see [Atomic Transactions](../core/atomic-transactions.md).</span></span>

>  <span data-ttu-id="ed70f-163">Vous pouvez améliorer les performances en réduisant le nombre de points de persistance dans une orchestration.</span><span class="sxs-lookup"><span data-stu-id="ed70f-163">You can improve performance by minimizing the number of persistence points in an orchestration.</span></span> <span data-ttu-id="ed70f-164">Vous pouvez regrouper **envoyer** point de formes dans une transaction atomique pour produire une persistance unique pour tous les **envoyer** formes.</span><span class="sxs-lookup"><span data-stu-id="ed70f-164">You can group **Send** shapes in an atomic transaction to produce a single persistence point for all of the **Send** shapes.</span></span> <span data-ttu-id="ed70f-165">L'arrêt simultané des étendues de transactions imbriquées produit un seul point de persistance pour les transactions.</span><span class="sxs-lookup"><span data-stu-id="ed70f-165">Ending nested transaction scopes at the same time produces a single persistence point for the transactions.</span></span>
