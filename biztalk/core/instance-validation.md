# <a name="instance-validation"></a><span data-ttu-id="cfd70-102">Validation de l’instance</span><span class="sxs-lookup"><span data-stu-id="cfd70-102">Instance Validation</span></span>
<span data-ttu-id="cfd70-103">L’Éditeur BizTalk appelle la **IInstanceValidator.ValidateInstance** méthode d’extension lorsque les conditions suivantes sont remplies :</span><span class="sxs-lookup"><span data-stu-id="cfd70-103">BizTalk Editor invokes the **IInstanceValidator.ValidateInstance** method of an extension when the following conditions are met:</span></span>  
  
-   <span data-ttu-id="cfd70-104">L’extension est définie comme norme à l’aide de la **Standard** propriété de la **schéma** nœud.</span><span class="sxs-lookup"><span data-stu-id="cfd70-104">The extension is set as standard by using the **Standard** property of the **Schema** node.</span></span>  
  
-   <span data-ttu-id="cfd70-105">Sur le **Pages de propriétés** boîte de dialogue associé au schéma, le **Type sortie générer l’Instance** est définie sur **natif.**</span><span class="sxs-lookup"><span data-stu-id="cfd70-105">On the **Property Pages** dialog box associated with the schema, the **Generate Instance Output Type** property is set to **Native.**</span></span>  
  
-   <span data-ttu-id="cfd70-106">Sur le **Pages de propriétés** boîte de dialogue associé au schéma, le **nom d’Instance d’entrée** est définie sur le nom d’un fichier contenant le message d’instance à valider.</span><span class="sxs-lookup"><span data-stu-id="cfd70-106">On the **Property Pages** dialog box associated with the schema, the **Input Instance Filename** property is set to the name of a file containing the instance message to be validated.</span></span>  
  
 <span data-ttu-id="cfd70-107">Le fichier spécifié dans le **nom d’Instance d’entrée** propriété est passée en tant que paramètre à la **IInstanceValidator.ValidateInstance** (méthode).</span><span class="sxs-lookup"><span data-stu-id="cfd70-107">The file specified in the **Input Instance Filename** property is passed as a parameter to the **IInstanceValidator.ValidateInstance** method.</span></span>  
  
 <span data-ttu-id="cfd70-108">En cas d’erreurs, les messages d’erreur sont retournées sous forme de tableau de **IValidationInfo** objets et s’affichent dans le [!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)] fenêtre liste des tâches.</span><span class="sxs-lookup"><span data-stu-id="cfd70-108">If errors occur, error messages are returned as an array of **IValidationInfo** objects, and are displayed in the [!INCLUDE[btsVStudioNoVersion](../includes/btsvstudionoversion-md.md)] Task List window.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="cfd70-109">Si le schéma contient des instructions de modèle et le fichier passé à la **ValidateInstance** méthode a été générée à l’aide correspondant **générer l’Instance** le message d’instance ne peut pas passer de la commande validation.</span><span class="sxs-lookup"><span data-stu-id="cfd70-109">If the schema contains pattern statements, and the file passed to the **ValidateInstance** method was generated by using the corresponding **Generate Instance** command, the instance message may not pass validation.</span></span>
